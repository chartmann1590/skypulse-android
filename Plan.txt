Build a complete native Android app called “SkyPulse” using Kotlin and Jetpack Compose.

Use the Stitch-generated UI/layout as the design reference. Recreate the screens, structure, spacing, colors, typography, and navigation as closely as possible.

App concept:
SkyPulse is a live aircraft tracking Android app. It uses only completely free open online data sources and must not require any paid API key, account login, or secret token.

Important rule:
Do NOT use FlightAware, FlightRadar24, AviationStack, Aviation Edge, RapidAPI, Google Maps paid APIs, or any paid/key-required aviation API. Do not scrape websites that block scraping or require login.

Primary live aircraft data:
Use ADSB.lol public API as the primary source.
Base docs/source:
- https://www.adsb.lol/
- https://api.adsb.lol/
- https://www.adsb.lol/docs/open-data/api/

Fallback live aircraft data:
Use OpenSky Network anonymous public REST access as fallback.
Base docs/source:
- https://opensky-network.org/
- https://openskynetwork.github.io/opensky-api/rest.html
- Anonymous access is allowed but rate-limited, so the app must cache and throttle requests.

Offline/local reference data:
Bundle OpenFlights airport and airline data as app assets.
Use it for airport search, airport names, IATA/ICAO lookup, city/country display, and airline name lookup where possible.
Source:
- https://openflights.org/data.php

Tech stack:
- Kotlin
- Jetpack Compose
- Material 3
- MVVM architecture
- Kotlin Coroutines
- Retrofit or Ktor client
- kotlinx.serialization or Moshi
- Room database for saved aircraft, saved airports, saved alert areas, and cached aircraft snapshots
- DataStore for settings
- OSMDroid or MapLibre/OpenStreetMap-based map tiles, not Google Maps paid API
- Local notifications for alerts
- Location permission support
- Minimum SDK: 26
- Target latest stable Android SDK

3. Live aircraft fetching
Implement a repository called AircraftRepository.

It should:
- Fetch aircraft by bounding box around user location
- Use ADSB.lol first
- Fall back to OpenSky if ADSB.lol fails
- Cache results for at least 15–30 seconds
- Never spam requests
- Handle no internet gracefully
- Handle malformed/missing data gracefully
- Show last updated time
- Expose data using StateFlow

Create normalized data model:

data class Aircraft(
    val id: String,
    val callsign: String?,
    val hex: String?,
    val latitude: Double?,
    val longitude: Double?,
    val altitudeFeet: Double?,
    val speedKnots: Double?,
    val headingDegrees: Double?,
    val verticalRate: Double?,
    val originCountry: String?,
    val lastSeenEpochSeconds: Long?,
    val source: DataSource
)

enum class DataSource {
    ADSB_LOL,
    OPENSKY,
    CACHE
}
